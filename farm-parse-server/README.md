# 农场变异代理模式使用说明

这个模式用于替代 Reqable 手动抓取 WebSocket HEX。启动后，手机或电脑把系统代理指向本服务，本服务只解析目标农场 WebSocket：

```text
wss://gate-obt.nqf.qq.com/prod/ws?
```

其它请求不会保存内容，也不会在页面展示。

## 启动服务

在 `farm-parse` 目录执行：

```powershell
npm install
npm run proxy
```

默认端口：

- 展示页面：`80`
- HTTP/HTTPS 代理：`8788`

本机打开：

```text
http://127.0.0.1/proxy.html
```

iOS 等局域网设备打开页面时，用启动日志或页面上显示的电脑 IP，例如：

```text
http://192.168.10.15/proxy.html
```

## 证书下载格式

服务现在提供三种下载地址，内容是同一张本地根证书，只是扩展名不同：

```text
http://127.0.0.1/ca.cer
http://127.0.0.1/ca.crt
http://127.0.0.1/ca.pem
```

推荐使用：

- Windows：`ca.cer`
- iOS：`ca.cer`

页面上的“下载根证书”按钮默认下载 `farm-parse-ca.cer`，Windows 和 iOS 都更容易直接打开安装。

## Windows 安装证书

1. 打开 `http://127.0.0.1/proxy.html`
2. 点击“下载根证书”，得到 `farm-parse-ca.cer`
3. 双击 `farm-parse-ca.cer`
4. 点击“安装证书”
5. 选择“本地计算机”
6. 如果弹出 UAC 管理员确认，点击“是”
7. 选择“将所有的证书都放入下列存储”
8. 点击“浏览”
9. 选择“受信任的根证书颁发机构”
10. 点击“确定”
11. 点击“下一步”
12. 点击“完成”
13. 如果出现安全警告，确认安装

如果你没有管理员权限，第 5 步也可以选择“当前用户”，但只对当前 Windows 用户生效。

## Windows 设置代理

1. 打开“设置”
2. 进入“网络和 Internet”
3. 进入“代理”
4. 打开“使用代理服务器”
5. 地址填写：

```text
127.0.0.1
```

1. 端口填写：

```text
8788
```

1. 保存
2. 打开农场页面或客户端
3. 回到 `proxy.html` 查看实时变异结果

使用结束后，建议关闭 Windows 代理。

## iOS 安装证书

1. 确保 iPhone 和运行代理服务的电脑在同一个 Wi-Fi
2. 在电脑的代理页面或启动日志中找到局域网 IP，例如 `192.168.10.15`
3. 用 iPhone 的 Safari 打开：

```text
http://电脑IP/proxy.html
```

示例：

```text
http://192.168.10.15/proxy.html
```

1. 点击“下载根证书”
2. iOS 会提示已下载描述文件
3. 打开“设置”
4. 进入“通用”
5. 进入“VPN 与设备管理”
6. 点击已下载的 `farm-parse-ca` 描述文件
7. 点击“安装”
8. 输入锁屏密码
9. 再次点击“安装”
10. 安装完成后返回“设置”
11. 进入“通用”
12. 进入“关于本机”
13. 进入“证书信任设置”
14. 找到 `farm-parse-ca`
15. 打开“完全信任”
16. 在弹窗中确认继续

iOS 只安装描述文件还不够，必须完成“证书信任设置”里的完全信任，否则 HTTPS/WSS 仍然无法解密。

## iOS 设置 Wi-Fi 代理

1. 打开“设置”
2. 进入“无线局域网”
3. 点击当前 Wi-Fi 右侧的信息按钮
4. 滚动到“HTTP 代理”
5. 选择“手动”
6. 服务器填写电脑 IP，例如：

```text
192.168.10.15
```

1. 端口填写：

```text
8788
```

1. 认证保持关闭
2. 打开农场
3. 回到电脑上的 `proxy.html` 查看实时变异结果

使用结束后，建议把 iOS 的 HTTP 代理改回“关闭”，并在不需要时删除证书。

## 删除证书

### Windows

1. 按 `Win + R`
2. 输入：

```text
certmgr.msc
```

1. 找到“受信任的根证书颁发机构”
2. 打开“证书”
3. 找到 `farm-parse` 或本代理生成的 CA 证书
4. 右键删除

如果安装到了“本地计算机”，可以运行：

```text
certlm.msc
```

然后在“受信任的根证书颁发机构”中删除。

### iOS

1. 打开“设置”
2. 进入“通用”
3. 进入“VPN 与设备管理”
4. 找到 `farm-parse-ca`
5. 删除描述文件

## 注意事项

- 不要把 `proxy-certs` 目录发给别人，里面包含本地 CA 私钥。
- 如果目标 App 对 `gate-obt.nqf.qq.com` 做了证书固定，即使安装根证书也可能抓不到。
- 使用结束后关闭系统代理，否则其它网络请求仍会经过本地代理。
